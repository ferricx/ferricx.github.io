Accessible Form Patterns

This application is a collection of common web form patterns built with accessibility in mind. Each page demonstrates a different layout approach while sharing a consistent set of accessibility features so that all users — including those relying on keyboard navigation or assistive technologies — can interact with forms effectively.

Form Patterns

  • Single Column

    A straightforward vertical form layout suitable for focused data-entry tasks such as account registration.

  • Wizard (Multi-step)

    Breaks a long form into discrete, labelled steps, reducing cognitive load and letting users track their progress.

  • Dialog

    Presents a form inside a modal dialog, keeping the user in context while capturing supplemental information.

  • Profile

    An editable profile page combining theme customisation controls with persistent user settings.

Accessibility Features

  • Skip navigation link — Lets keyboard users jump directly to the main content, bypassing the repeated header navigation.
  • Semantic landmark regions — <header>, <nav>, <main>, and <footer> give screen-reader users a clear page map.
  • ARIA labels and roles — Navigation regions, dialogs, alerts, and interactive widgets carry descriptive ARIA attributes so their purpose is announced correctly.
  • Live region error announcements — Validation errors use role="alert" or aria-live so they are surfaced to assistive technologies without a page reload.
  • Error summary component — A consolidated error list at the top of each form links directly to each invalid field, following WCAG 3.3.1 guidance.
  • Form group component — Wraps each field with a visible <label>, an optional hint, and an inline error message wired via aria-describedby.
  • State combobox — An accessible combobox that supports keyboard navigation, type-ahead filtering, and proper role="combobox" / role="listbox" semantics.
  • Popover tooltip — Uses the native Popover API with a focusable trigger so supplementary hints are reachable without a mouse.
  • Focus management in dialogs — Focus is moved into the dialog on open and returned to the trigger element on close.
  • Active route indication — The current navigation link receives an active class and contrasting style so the user always knows their location.
  • Color contrast and theming — A CSS custom-property theming system ensures sufficient contrast ratios are maintained across light and dark modes.
